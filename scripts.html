Command line utility: ecl2csv¶

From the command line there is a utility for dumping the data of interest to CSV files.

usage: ecl2csv [-h]
               {grid,smry,nnc,faults,rft,satfunc,compdat,equil,gruptree,wcon}
               ...

Sub-commands:¶

grid¶

Extract grid data

ecl2csv grid [-h] [--initkeys INITKEYS [INITKEYS ...]] [--rstdate RSTDATE]
             [-o OUTPUT] [--dropconstants] [-v]
             DATAFILE

Positional Arguments¶

DATAFILE

Name of Eclipse DATA file. INIT and EGRID file must lie alongside.

Named Arguments¶

--initkeys

INIT vector wildcards for vectors to include

Default: “*”

--rstdate

Point in time to grab restart data from, either ‘first’ or ‘last’, or a date in YYYY-MM-DD format

Default: “”

-o, --output

Name of output csv file.

Default: “eclgrid.csv”

--dropconstants

Drop constant columns from the dataset

Default: False

-v, --verbose

Be verbose

Default: False

smry¶

Extract summary data

ecl2csv smry [-h] [--time_index TIME_INDEX]
             [--column_keys COLUMN_KEYS [COLUMN_KEYS ...]] [-o OUTPUT] [-v]
             DATAFILE

Positional Arguments¶

DATAFILE

Name of Eclipse DATA file. UNSMRY file must lie alongside.

Named Arguments¶

--time_index
Time resolution mnemonic; raw, daily, monthly or yearly.

Data at a given point in time applies until the next point in time. If not raw, data will be interpolated. Use interpolated rate vectors with care. Default is raw, which will include clock times.

Default: “raw”

--column_keys
Summary column vector wildcards, space-separated.

Default is to include all summary vectors available.

-o, --output

Name of output csv file. Use ‘-‘ to write to stdout. Default ‘summary.csv’

Default: “summary.csv”

-v, --verbose

Be verbose

Default: False
